## 1. 产品概述

### 1.1 产品背景

Qcode 当前是一个纯 CLI Coding Agent，虽然功能完整，但在以下场景存在体验痛点：

- **流式输出不够直观**：SSE 文本流在终端中与工具日志混排，难以清晰追踪对话流
- **多 Agent 协作缺乏可视化**：teammate 状态、任务看板、协议流程需要通过命令查看，不直观
- **长对话难以回顾**：compaction 后的历史记录在终端中分散，难以快速定位关键信息
- **工具日志堆叠**：多工具调用时，命令行输出混乱，难以追踪调用链路

### 1.2 产品目标

将 Qcode 的 CLI 体验升级为 **可视化 Web 界面**，核心目标是：

- ✅ **提升对话体验**：流式输出可视化，工具调用时间线清晰展示
- ✅ **增强协作可见性**：teammate 状态、任务看板、协议审批实时展示
- ✅ **优化长对话管理**：compaction 提示清晰，历史回顾便捷
- ✅ **提升配置易用性**：模型配置、连接状态可视化设置

### 1.3 目标用户

| 用户类型 | 使用场景 | 核心需求 |
|---------|---------|---------|
| **开发者** | 使用 Qcode 进行代码辅助开发 | 清晰的对话流、工具调用追踪、快速代码查看 |
| **技术团队** | 使用多 Agent 协作功能 | 实时查看 teammate 状态、任务进度、协议审批 |
| **项目管理者** | 需要查看任务图和团队状态 | 任务看板、团队协作总览、进度跟踪 |

## 2. 核心功能模块

### 2.1 对话主页面

**功能描述**：作为产品核心入口，承载用户与 Agent 的对话交互、流式输出、工具调用链路展示。

**核心需求**：

| 功能点 | 描述 | 优先级 |
|-------|------|-------|
| **SSE 流式输出** | 模型响应逐 token 渲染，不等整段完成才显示 | P0 |
| **消息时间线** | 用户消息、助手消息、工具消息混排展示 | P0 |
| **工具调用卡片** | 显示 `tool_call`，包含工具名、参数预览、运行中状态 | P0 |
| **工具结果卡片** | 显示 `tool_result`，包含执行结果、耗时、成功/失败态 | P0 |
| **成对展示** | `tool_call` 与 `tool_result` 必须成对显示，不乱序 | P0 |
| **Compaction 提示** | 触发压缩后显示"已压缩上下文"提示块 | P1 |
| **Transcript 入口** | 提供 transcript 文件路径引导或复制入口 | P1 |
| **错误提示** | 请求失败、超时、工具执行失败都有明确错误态 | P0 |
| **Markdown 渲染** | 支持 Markdown 格式渲染（代码块、列表、标题） | P1 |

**界面原型（文字描述）**：


### 2.2 任务看板

**功能描述**：可视化展示持久任务图，支持状态切换、依赖关系查看。

**核心需求**：

| 功能点 | 描述 | 优先级 |
|-------|------|-------|
| **四列布局** | Ready / In Progress / Blocked / Completed 四列 | P0 |
| **任务卡片** | 显示 subject、owner、requiredRole、状态 | P0 |
| **实时刷新** | 任务状态变化后自动刷新看板 | P0 |
| **状态切换** | 支持拖拽或点击切换任务状态 | P1 |
| **依赖关系** | 显示 blockedBy 关联，可跳转到依赖任务 | P1 |
| **空状态兜底** | 空任务时显示引导文案 | P2 |
| **任务详情** | 点击任务卡片展开完整 JSON 或详细描述 | P1 |

**数据结构**：

```json
{
  "task_id": 1,
  "subject": "验证登录流程",
  "description": "...",
  "status": "pending",  // pending, in_progress, completed
  "owner": "tester",
  "requiredRole": "tester",
  "blockedBy": [2, 3]
}
```

## 5. 验收标准

### 5.1 功能验收

| 编号 | 验收项 | 验收方法 |
|------|-------|---------|
| **F1** | SSE 流式输出正确渲染 | 手动测试：输入问题，观察逐 token 出现 |
| **F2** | 工具调用与结果成对展示 | 手动测试：触发 `read_file` 等工具，验证成对显示 |
| **F3** | 任务看板四列正确分类 | 集成测试：创建不同状态任务，验证归类正确 |
| **F4** | Teammate 状态实时更新 | 集成测试：teammate 状态变化，UI 实时刷新 |

## 6. 技术建议

### 6.1 推荐技术栈

| 层级 | 推荐技术 | 说明 |
|------|---------|------|
| **前端框架** | React / Vue | 选择团队熟悉的技术栈 |
| **状态管理** | Zustand / Pinia | 轻量级，适合中小型应用 |
| **UI 组件库** | shadcn/ui / Element Plus | 快速构建界面 |
| **SSE 处理** | EventSource API / fetch with reader | 原生支持 |
| **测试框架** | Vitest + Testing Library + Playwright | 符合测试计划要求 |
| **构建工具** | Vite | 快速开发体验 |

### 6.2 关键技术点

1. **SSE 流式处理**：使用 `EventSource` 或 `fetch` + `ReadableStream` 处理 SSE 流
2. **WebSocket 实时推送**： teammate 状态、任务变化可通过 WebSocket 实时推送
3. **Compaction 检测**：监听 SSE 中的 compaction 事件，插入提示块
4. **Markdown 渲染**：使用 `react-markdown` 或类似库
5. **状态持久化**：配置和会话状态可持久化到 LocalStorage
